### A2A endpoints and version upgrade options in 8.11.0

Hi Elastic team,

I'm building a multi‑agent system for the Agent Builder Hackathon using version 8.11.0 (serverless). I've noticed that A2A endpoints sometimes return a valid Agent Card but often give 404. Other agents consistently return 404. Could you clarify:

\- Is A2A intended to work in version 8.11.0, or is it only available from 9.2 onward as the docs suggest?

\- If A2A requires a newer version, is there a way to upgrade my serverless deployment to 9.x? If not, would you recommend sticking with direct tool orchestration for the hackathon?

Thanks for your guidance!

Serverless is always on the latest version (often even a little ahead of the latest release). The 8.11 is there for backwards compatibility and it's not the real version (there's no version for Serverless). So your version is good.
